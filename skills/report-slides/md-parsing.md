# Markdown 到幻灯片解析规则

将 Markdown 内容自动解析为幻灯片结构的映射规则。

## 基础映射

| Markdown 元素 | 幻灯片映射 | 说明 |
|---------------|-----------|------|
| `# 一级标题` | 封面页标题 | 整个文档只有一个封面 |
| `## 二级标题` | 新幻灯片的标题 | 每个 `##` 开始一张新幻灯片 |
| `### 三级标题` | 幻灯片内的子标题/分组标题 | 不会创建新幻灯片 |
| `- 列表项` | 要点/卡片内容 | 根据数量自动选择版式 |
| `1. 有序列表` | 步骤/时间线 | 自动添加序号样式 |
| `> 引用块` | 总结/金句页 | 单独成页，大字号居中 |
| `---` 分隔线 | 强制分页 | 在同一个 `##` 内强制拆分 |
| `**加粗文本**` | 强调文本 | 使用强调色 |
| 表格 | 数据表格/对比表 | 自动适配为卡片或表格 |
| 图片 `![](path)` | 图片幻灯片 | 单独成页或嵌入内容页 |

## 智能版式识别

### 关键指标识别

当列表项中包含**数字 + 单位/描述**的模式时，自动识别为关键指标：

```markdown
## 关键成果
- **4000万+** 助力续费
- **3000人** 开发者话题群
- **581** 服务客户
- **~200 ⭐** GitHub Stars
```

识别规则：
- 加粗的数字（含中文单位如"万"、"亿"）
- 数字后跟 `+`、`%`、`⭐` 等符号
- 数字后跟中文单位（人、个、万、亿等）

映射为**数据看板**版式：大字号数字 + 小字号描述标签。

### 并列主题识别

当一个 `##` 下有 2-4 个 `###` 子标题，每个子标题下有简短描述时，识别为并列主题：

```markdown
## 核心结果
### 产品体验升级
围绕主链路持续治理...

### AI 平台化
AI 从单点功能进入生态...

### 开发者生态
社区发布、模板贡献...

### 商业落地
签约客户、营收验证...
```

映射为**四象限卡片**版式。

### 策略/支柱识别

当一个 `##` 下有恰好 3 个 `###` 子标题时，优先识别为三列策略布局：

```markdown
## FY27 核心策略
### 新方式
AI 驱动生成式构建
- Vibe Coding
- AI Skill

### Back to Developers
Reshape platform-developer relationship
- OpenAPI
- MCP Server

### 新市场
私有化与国际化双轮驱动
- 私有化
- 国际化
```

映射为**三列策略**版式。

### 时间线识别

有序列表中包含时间/日期关键词时，识别为时间线：

```markdown
## 里程碑
1. **Q1** 完成基础架构搭建
2. **Q2** 发布 MVP 版本
3. **Q3** 用户规模突破 1000
4. **Q4** 实现商业化闭环
```

映射为**时间线**版式。

## 内容密度控制

每张幻灯片的内容有严格上限。解析时如果内容超出限制，必须自动拆分：

| 版式 | 最大容量 | 超出处理 |
|------|----------|----------|
| 封面页 | 1 标题 + 1 副标题 | 多余内容移到下一页 |
| 内容页 | 4-6 个要点 | 拆分为"（续）"页 |
| 数据看板 | 8 个指标 | 拆分为两个看板页 |
| 卡片网格 | 4 张卡片 | 拆分为两页 |
| 三列策略 | 3 列 | 不可超出 |
| 时间线 | 4-6 个节点 | 拆分为两页 |

## 元信息提取

从 Markdown 的 frontmatter 或首段提取元信息：

```markdown
---
title: FY26 年度总结
author: Your Name
team: Your Team
date: 2026.03
---
```

或从正文首段提取：
```markdown
# FY26 年度总结与 FY27 战略思考
Your Name ｜ Product Team / Annual Report
2026.03 · Product Team
```

提取的元信息用于封面页的副标题和元信息行。

## 章节分组

如果 Markdown 中有明确的章节结构（如"第一部分"、"Part 1"、"FY26 回顾"、"FY27 展望"），自动生成章节分隔页：

```markdown
## 📋 FY26 年度回顾
```

带 emoji 前缀的 `##` 标题生成章节分隔页（大标题居中 + emoji 装饰），而非普通内容页。

## 特殊标记

| 标记 | 效果 |
|------|------|
| `<!-- speaker: 备注内容 -->` | 生成为 HTML 注释（演讲者备注） |
| `<!-- style: apple -->` | 指定使用 Apple 风格 |
| `<!-- split -->` | 等同于 `---`，强制分页 |
| `🎯` `📊` `🚀` 等 emoji | 用作卡片/列表项的图标 |
